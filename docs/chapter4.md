### 为什么接单是最快的变现方式？

和内容创作相比，接单有一个巨大的优势——**钱来得更直接**。

| 维度 | 内容创作 | 接单 |
|------|---------|------|
| 见钱时间 | 1-3个月 | 2-3周 |
| 单笔收入 | 几十到几百 | 几千到几万 |
| 确定性 | 低（靠流量） | 高（谈好了就给） |
| 技能要求 | 写作能力 | 编程能力 |
| 时间投入 | 持续产出 | 项目制，做完就收 |

**但传统接单有个致命问题**：报价按工作量算，时薪低得可怜。

一个普通的外包项目：
- 客户预算：¥8,000
- 传统耗时：40小时（5天）
- 实际时薪：**¥200/h**

用AI辅助后同样这个单子：
- AI辅助耗时：10小时（1.5天）
- 同样报价：¥8,000
- 实际时薪：**¥800/h**

**同样的技术能力，时薪翻了4倍。这就是AI时代的接单逻辑。**

### 4.2 怎么报价？大多数人都报错了

#### 三种报价策略

**策略一：按时计费（新手期）**

适用于：还不确定自己效率的时候。

```
你的市场时薪 = （你当前月薪 ÷ 22 ÷ 8）× 2
```

举例：月薪15,000 → 时薪 = (15000÷22÷8)×2 = **¥170/h**

为什么乘2？因为自由职业没有社保公积金、没有带薪休假、不稳定。**你的实际成本远高于工资。**

**策略二：按项目报价（进阶期）**

适用于：对工作量有判断力之后。

公式：
```
报价 = 你的时薪 × 预估小时数 × 1.3（风险系数）
```

关键：**永远不要按你的实际耗时报价，要按市场价值报价。** 如果AI帮你10小时干完了一个传统需要40小时的活，你报的是40小时的价值，不是10小时的。

**策略三：按价值报价（高手期）**

适用于：你有品牌/口碑之后。

不报"做这件事要多少时间"，而是报"这件事值多少钱"。客户要的不是你的时间，是结果。

一个真实的例子：
- 帮一个小电商做了个自动化订单处理系统
- 实际开发时间：3天（AI辅助）
- 给客户带来的价值：每月省200人工工时 = ¥20,000/月
- 最终报价：¥50,000（一次性）
- 客户的反应：**立刻成交**（因为对他来说3个月就回本了）

#### 报价的黄金法则

1. **第一次报价宁可偏高也不要偏低** — 低了后面很难涨，高了可以谈
2. **永远留出20%的缓冲** — 总会有你没预料到的需求变更
3. **分阶段收款** — 30%启动 + 50%交付 + 20%验收（防止跑单）
4. **把维护费用单独算** — "上线后3个月内免费Bug修复，之后按¥XXX/h"

### 4.3 AI辅助接单的全流程

#### Phase 1：需求分析（原来2小时 → 现在30分钟）

大多数程序员在这步浪费大量时间。客户说"做个网站"，你需要搞清楚：

**必须问清楚的10个问题：**
1. 目标用户是谁？（决定UI风格和技术选型）
2. 核心功能是什么？（区分MVP和锦上添花）
3. 有没有参考产品？（直接看竞品最省事）
4. 数据量级大概多少？（决定技术方案）
5. 是否需要后台管理？
6. 预算范围？（筛选客户的重要标准）
7. 时间要求？（评估可行性）
8. 是否有设计稿？（影响前端工作量）
9. 部署在哪里？（云服务器/自有服务器）
10. 后续是否需要维护？

**用AI加速的方法**：
- 把客户的原始需求描述丢给AI："请帮我分析这个需求，列出所有功能点、可能的技术风险、以及我还需要确认什么"
- 让AI生成需求文档初稿
- 你来审核和补充

#### Phase 2：技术方案（原来4小时 → 现在45分钟）

让AI帮你做技术选型：

```
Prompt示例：
"我要做一个XX类型的Web应用，有以下特点：[列举]。
请推荐技术栈（前后端+数据库+部署），并说明理由。
要求：开发速度快、维护成本低、适合1-2人团队。"
```

然后让AI生成项目脚手架。Cursor/Windsurf这类AI编辑器可以直接从需求描述生成初始代码结构。

#### Phase 3：编码实现（原来20小时 → 现在5-7小时）

这是AI提升最大的环节：

| 任务类型 | 传统耗时 | AI辅助耗时 | 提升倍数 |
|---------|---------|-----------|---------|
| CRUD增删改查页面 | 4h | 0.5h | 8x |
| API接口开发 | 3h | 0.5h | 6x |
| 表单验证+错误处理 | 2h | 0.3h | 7x |
| 数据库设计和迁移 | 2h | 0.5h | 4x |
| 第三方API对接 | 3h | 1h | 3x |
| UI页面搭建 | 4h | 1h | 4x |
| 测试+调试 | 2h | 1h | 2x |

**注意**：不是所有代码都能交给AI。以下部分你必须亲自把关：
- **架构设计** — AI不懂业务上下文
- **安全相关** — 认证授权、数据校验
- **核心业务逻辑** — 这是你被雇佣的原因
- **代码审查** — AI生成的代码必须过你的眼睛

#### Phase 4：交付验收（原来2小时 → 现在30分钟）

- 用AI自动生成部署文档
- 用AI生成操作手册（给非技术客户看的）
- 用AI准备演示脚本（怎么展示最好看）

### 4.4 接单的5个坑

**坑1：需求蔓延（Scope Creep）**
> "再加个小功能呗，很快的"

应对：所有需求变更走正式确认，小的也要邮件/消息记录。"可以加，但这属于二期，我们先把当前版本交付完。"

**坑2：预付款不要就不开始**
> "项目结束一起结"

应对：坚持30%预付款规则。如果客户不同意，大概率会赖账。这种客户不做也罢。

**坑3：低价竞争陷阱**
> 别人报价5000，你报3000想抢单

应对：**低价抢来的客户是最差的客户**。他们预期低、要求多、后续麻烦。找到认可你价值的客户，而不是拼价格的客户。

**坑4：过度承诺**
> 为了拿单什么都答应

应对：诚实告知风险和时间。"这个功能可以做，但会增加大约X天的工期，您看要不要调整优先级？" 专业=靠谱，不等于什么都答应。

**坑5：不签合同/不留书面记录**
> "咱们口头说好的"

应对：哪怕只有几千块的单子，也必须有书面协议。微信聊天记录截图都算，但不能什么都不留。

### 4.5 从接单到建立被动收入

接单的本质还是**卖时间**。长期来看，你应该把接单中学到的东西变成产品：

```
第1阶段：纯接单（卖时间）
    ↓ 发现某些需求反复出现
第2阶段：标准化服务（卖SOP）
    ↓ 把常见需求做成固定套餐
第3阶段：产品化（卖产品）
    ↓ 直接卖工具/模板/课程
```

很多成功的独立开发者都是这条路走过来的：
- 先帮客户做了10个官网 → 做了个网站生成器SaaS
- 先写了100篇技术文章 → 出了一本技术写作教程
- 先帮人配了50次服务器 → 做了个一键部署工具

**接单不只是赚钱，更是市场调研。每一个单子都在告诉你市场上缺什么、人们愿意为什么付费。**
