### 5.1 什么是"微产品"？

微产品 = **解决一个具体问题的小工具/数字产品**，不是庞大的SaaS平台。

好的微产品特征：
- ✅ 解决一个明确的痛点
- ✅ 一个人（+AI）能在2-4周内做出MVP
- ✅ 可以用一句话讲清楚是做什么的
- ✅ 目标用户能自己找到你
- ✅ 定价在 $5-50 或 ¥29-299 区间

**反例（不是好微产品）**：
- ❌ "一个全能的企业管理ERP系统" — 太大、太复杂
- ❌ "一个比微信更好的聊天App" — 不可能一个人做出来
- ❌ "AI驱动的全自动化营销平台" — 需要团队和大量资金

### 5.3 从想法到上线：4步走

#### Step 1：验证需求（2-3天）

在写一行代码之前，先确认真的有人愿意为这个方案付钱：

**最低成本验证方法**：
1. 在相关社区发帖描述你要做的产品，看反应
2. 在Product Hunt / V2EX / Hacker News 发个预告帖
3. 找5-10个目标用户聊，问他们："如果这个工具存在，你会用吗？愿意付多少钱？"
4. 搜一下有没有竞品 → 如果有，看他们的评价里用户在抱怨什么（那就是你的机会）

**危险信号**：
- 说"挺好的"但不愿意给邮箱 → 不是真需求
- 问了一圈发现没人主动找过类似解决方案 → 可能是你自嗨
- 你自己不需要这个东西，身边人也不需要 → 危险

#### Step 2：做出MVP（1-2周）

MVP = Minimum Viable Product（最小可行产品）

**MVP只包含核心功能，其他都是噪音：**

| 必须有的 | 可以后来加的 |
|---------|------------|
| 核心功能跑通 | 用户系统/登录注册 |
| 一个清晰的购买流程 | 后台管理面板 |
| 基础的支付能力 | 数据分析/报表 |
| 一个落地页 | 多语言支持 |
| | 移动端适配 |

**AI加速MVP开发的实战组合**：
```
前端页面：v0.dev / Bolt.new / Cursor（30分钟出原型）
后端API：Cursor AI / Claude Code（让AI根据你的描述直接生成）
数据库：Supabase（自带Auth+DB+实时订阅，免费起步）
部署：Vercel（前端）/ Railway / Fly.io（后端）
支付：Lemon Squeezy（处理全球支付+税务）
```

这个组合可以让一个人在1-2周内做出一个功能完整的MVP。**这在5年前至少需要2-3个人的团队干2个月。**

#### Step 3：发布上线（2-3天）

**发布清单**：
- [ ] 落地页上线（说明产品是什么、解决什么问题、价格多少）
- [ ] 支付链路测试（真实下单走一遍）
- [ ] Product Hunt 发布准备（文案+截图+视频）
- [ ] 准备3篇推广用的内容（教程/案例/使用技巧）
- [ ] 社交媒体账号准备好（Twitter/X、即刻等）

**发布日策略**：
- 选周二到周四（避免周一忙周五不想看）
- 早上8-10点（美国太平洋时间）或晚上8-10点（北京时间）
- 同时在多个渠道发布
- 自己先准备好第一批评论/互动（找朋友帮忙也行）

### 5.4 定价策略

数字产品的定价是一门学问。太低=不赚钱+低端形象，太高=没人买。

#### 经验证的定价区间

| 产品类型 | 免费版 | 付费版 | 说明 |
|---------|--------|--------|------|
| 小工具/插件 | 基础功能免费 | $5-19/月 | Freemium模式 |
| 模板/资源包 | 无 | $19-49 一次性 | 买断制 |
| SaaS（轻量） | 有限使用免费 | $9-29/月 | 订阅制 |
| 专业工具 | 试用14天 | $29-99/月 | 企业向 |

#### 一个有效的定价技巧：锚定效应

不要只放一个价格选项。放三个：

| 方案 | 价格 | 包含 |
|------|------|------|
| 个人版 | ¥99/年 | 基础功能，够个人用 |
| **Pro版（推荐）** | **¥199/年** | **全部功能+优先支持** |
| 团队版 | ¥499/年 | Pro全部 + 团队协作 + API |

大多数人会选中间那个。这不是偶然，是心理学。

### 5.5 真实案例拆解

### 案例1：tinywow.com
- **做什么**：各种在线小工具（PDF转换、图片压缩、文字处理等）
- **怎么开始的**：一个人做了几个实用的小工具放在网上
- **现在**：月访问量1000万+，主要靠广告收入
- **关键成功因素**：工具多而全、SEO做得极好、完全免费靠广告变现

### 案例2：photoai.com
- **做什么**：AI生成人物照片
- **启动**：一个人用现有AI模型包装了一个简单的Web界面
- **收入**：首月$15K，后来被收购
- **关键成功因素**：踩中了AI摄影的热点、产品极简（一个功能做到极致）、传播性强

### 案例3：resumake.io（国内同类：超级简历）
- **做什么**：在线简历制作工具
- **启动**：一个前端开发者觉得市面工具都太难用
- **现在**：通过Pro版+模板销售盈利
- **关键成功因素**：解决了真实的痛点、开发者出身所以体验做得好

**这三个案例的共同点**：
1. 从一个小而具体的问题开始
2. MVP非常简单，后续逐步迭代
3. 创始人自己就是目标用户（"狗粮"原则）
4. 都是先有了用户再想怎么变现
