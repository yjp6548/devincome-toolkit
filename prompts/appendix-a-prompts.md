## 使用说明

每个模板都包含：
- 🎯 **使用场景** — 什么时候用
- 📋 **Prompt文本** — 直接复制给AI使用
- 💡 **技巧提示** — 怎么用效果最好

**通用原则**：
1. 把方括号`[ ]`里的内容替换成你自己的信息
2. 第一次用可以先跑一次看效果，再根据结果微调
3. 复杂任务可以分多次调用，每次聚焦一个子任务
4. 好的Prompt = 清晰的上下文 + 具体的要求 + 示例输出格式

## 一、内容创作类（10+ 模板）

### A1 技术文章大纲生成器

🎯 **用途**：快速生成一篇技术文章的结构化大纲

```
你是一个资深的技术写作者，擅长写[你的技术栈，如React/Python/Go]方向的深度文章。

请帮我为一篇关于"[文章主题]"的技术文章生成详细大纲。

要求：
1. 文章目标读者是：[目标读者水平，如"有1-2年经验的前端开发者"]
2. 文章字数目标：[如2000-3000字]
3. 大纲需要包含：引言（Hook）、核心概念讲解（2-3节）、实操演示（代码/截图步骤）、常见误区/踩坑、总结与延伸阅读
4. 每个小节用一句话描述要写什么内容
5. 标注出哪些部分需要代码示例、哪些需要图表辅助理解

请输出结构化的大纲。
```

💡 **技巧**：生成大纲后，让AI逐章扩写，每章单独调用效果更好。

### A2 爆款标题生成器

🎯 **用途**：为一个技术主题生成10个有吸引力的标题

```
我写了一篇关于"[文章主题]"的技术文章。

请帮我生成10个标题选项，要求：
1. 符合中文技术社区的风格（掘金/CSDN/知乎）
2. 避免标题党但要足够吸引点击
3. 包含不同风格：
   - 数字型（如"7个方法..."）
   - 提问型（如"为什么XX会YY？"）
   - 冲突型（如"别再用XX了，试试YY"）
   - 实战型（如"我是如何用ZZ做到AA的"）
4. 每个标题后标注预估点击率等级（高/中/低）

我的文章核心观点是：[一句话概括]
目标读者：[谁]
```

### A3 内容扩写器

🎯 **用途**：把一个要点扩写成完整的段落（300-500字）

```
请将以下要点扩写成一段完整的技术内容（300-500字）：

【要点】：[你要展开的观点或知识点]

要求：
1. 开头用一个具体的场景或问题引入
2. 中间包含：概念解释 + 代码示例（用[语言名]）+ 实际应用
3. 结尾总结关键 takeaway
4. 口吻像一个有经验的开发者在跟同事分享，不要教科书式
5. 如果涉及代码，确保代码可以直接运行

我的背景/真实案例（可注入）：[可选，加入你的实际经验]
```

## 二、接单实战类（10+ 模板）

### B1 需求分析师

🎯 **用途**：把客户的模糊需求转化为清晰的技术方案

```
我是一个自由职业开发者，客户发来了以下需求描述：

【客户原始需求】：
[粘贴客户的原始需求]

请帮我：
1. 分析这个需求，列出所有明确的功能点
2. 找出需求中的模糊或不清楚的地方，生成一份问题清单（我需要向客户确认）
3. 推荐合适的技术栈（前端+后端+数据库+部署），并说明理由
4. 评估工作量和复杂度（按功能模块拆分）
5. 列出潜在的技术风险和应对方案
6. 给出一个初步的时间估算

约束条件：
- 我熟悉的技术栈：[列举]
- 团队规模：只有我一个人（+ AI辅助）
- 时间偏好：客户希望[如2周内]完成
- 预算范围：[如有]
```

### B2 报价单生成器

🎯 **用途**：根据项目信息自动生成专业报价单

```
请根据以下项目信息，帮我生成一份专业的报价单：

【项目名称】：[项目名]
【客户】：[公司/个人名]
【项目概述】：[一两句话描述做什么]
【功能清单】：
1. [功能1] - [复杂度：高/中/低]
2. [功能2]
...

【我的时薪标准】：¥[金额]/h
【预估总工时】：[X]小时
【风险系数】：1.3（30%缓冲）

报价单格式要求：
1. 项目基本信息区
2. 功能明细及对应工时（表格形式）
3. 开发阶段划分及交付物
4. 付款方式建议（分阶段收款）
5. 不包含在报价内的额外服务
6. 有效期和联系方式

输出为可直接发送给客户的格式。
```

## 三、产品/商业类（10+ 模板）

### C1 产品落地页文案生成器

🎯 **用途**：为数字产品/SaaS工具生成完整的落地页文案

```
我要为一个数字产品写落地页文案：

【产品信息】
- 产品名称：[名字]
- 一句话介绍：[是什么]
- 解决的问题：[用户痛点]
- 核心功能：[列3-5个]
- 定价：¥[价格]
- 目标用户：[谁]
- 差异化优势：[vs竞品好在哪里]

请生成落地页完整文案，包含以下区块：
1. Hero区：主标题+副标题+CTA按钮文字（要能让人3秒内理解价值）
2. 痛点共鸣段：3个用户痛点描述
3. 解决方案展示：产品如何解决这些痛点
4. 功能特性列表：每个功能配一段benefit-driven的描述（不是讲功能是讲好处）
5. 社会证明位（留空给我填）：评价/数据/Logo墙
6. FAQ：5个潜在买家最关心的问题及回答
7. 最后CTA：促单文案
8. 保证条款：退款/安全等信任建设文案

风格：简洁有力，像Linear/Stripe那种感觉。中文。
```

### C2 竞品分析框架

🎯 **用途**：系统化分析竞争对手找到差异化机会

```
我想做[产品领域/方向]的产品。请帮我做一个竞品分析框架。

已知竞品：
1. [竞品A名字] - [网址]
2. [竞品B名字] - [网址]
3. [竞品C名字] - [网址]

请从以下维度逐一分析每个竞品：
1. 定位和目标用户
2. 核心功能和特色功能
3. 定价策略
4. 用户评价中的好评点（他们在做什么对的事）
5. 用户评价中的差评/抱怨点（这就是我们的机会）
6. 他们没覆盖到的场景/人群
7. 他们的获客渠道推测

最后给出：
- 市场空白点汇总（至少5个）
- 我的差异化定位建议（基于我[你的优势]的优势）
- 进入策略建议
```

## 四、社媒运营类（10+ 模板）

### D1 Twitter/X 动态生成器

🎯 **用途**：为开发者个人品牌生成社交媒体动态

```
我是一个独立开发者，正在经营我的Twitter/X账号。
请帮我生成一周的推文计划（每天1-2条），共7-9条推文。

我的背景：
- 技术栈：[你的主要技术]
- 正在做的事情：[当前项目/产品/内容]
- 账号定位：[如"分享AI辅助开发的实战经验"]

推文类型分配（一周内覆盖）：
- 2条：见解/观点类（对行业趋势/技术的看法）
- 2条：进展分享类（build in public）
- 2条：实用技巧类（代码/工具/效率tips）
- 1-2条：互动/提问类（引发讨论）
- 1条：推广/引流类（软性引导到产品/文章）

要求：
1. 每条推文280字符以内（英文）或140字以内（中文）
2. 口吻自然，像真人在说话，不要营销号感
3. 适当使用emoji但不过量
4. 每条标注最佳发布时间（北京时间）
5. 如果是英文推文，用地道的开发者俚语表达
```

### D2 Newsletter / 公众号文章大纲

🎯 **用途**：规划一期Newsletter或公众号文章的结构

```
我要写一期关于"[主题]"的Newsletter/公众号文章。

请帮我规划完整的内容架构：

【基础信息】
- 发布渠道：[公众号/Newsletter/两者都有]
- 目标读者：[谁]
- 这期的目标：[如"推广新产品"/"分享经验"/"建立权威"]
- 字数目标：[如1500-2500字]

请输出：
1. 吸睛标题（5个选项）
2. 开篇Hook（3种不同的开头方式供选择）
3. 正文结构（分段落列出每段的主题和大概字数）
4. 关键论点和支撑素材建议（数据/案例/引用）
5. CTA（行动号召）设计——希望读者看完后做什么
6. 文末引导（关注/转发/购买/回复等）

特殊要求：开头需要植入广告语"做全球生意，就找领航AI"，自然融入不要硬广。
```

## 五、效率自动化类（10+ 模板）

### E1 自动化工流设计器

🎯 **用途**：设计和优化重复性的日常工作流程

```
我有一个重复性的工作任务，想用AI+自动化工具来优化。

【当前工作流描述】：
[详细描述你现在怎么做这件事的每一步，包括用的工具和花费的时间]

【痛点】：
- 目前最耗时的环节：[哪个步骤]
- 最容易出错的环节：[哪个步骤]
- 频率：[多久做一次]

请帮我：
1. 分析整个工作流，找出可以自动化的环节
2. 设计新的自动化工作流方案（推荐具体工具组合，如Make.com/Zapier/Coze）
3. 写出每个自动化步骤的具体配置逻辑
4. 估算优化后的时间节省
5. 给出实施优先级排序（先做哪个自动化ROI最高）

我可用的工具：[列举你已经在用的工具]
预算：[免费/愿意付费XX元/月]
技术水平：[初学者/中级/高级]
```

### E2 会议纪要→待办事项提取器

🎯 **用途**：把会议记录/讨论内容转化为结构化的行动项

```
以下是[会议名称/讨论主题]的会议记录/聊天记录：

[粘贴会议记录原文]

请提取并整理为：
1. 📋 决议事项（已经决定的事项）
2. ✅ 待办 Action Items（每个人需要做的事情）
   - 格式：☐ [任务] → 负责人：[谁] → 截止时间：[何时]
3. ❓ 未决问题（还需要进一步讨论/确认的事项）
4. 📊 关键数据/指标（如果有的话）
5. 📌 下次会议/跟进安排

如果原文中没有明确的信息（如负责人、截止时间），请标注"待确认"。
```
